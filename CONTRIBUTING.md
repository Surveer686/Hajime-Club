### 1. Fork the Repository
Click the **Fork** button on GitHub and clone your fork locally.

```
git clone https://github.com/your-username/hajime-club-website.git
````

---

### 2. Create a Feature Branch

Always work on a new branch.

```
git checkout -b feature/your-feature-name
```

---

### 3. Follow Coding Standards

* Write clean, readable, and well-documented code
* Follow the existing folder and file structure
* Use meaningful variable, function, and commit names
* Prefer reusable components and custom hooks
* Ensure accessibility (labels, ARIA attributes, keyboard navigation)
* Keep UI responsive and mobile-friendly

---

### 4. Install Dependencies & Run Locally

```
npm install
npm run dev
```

Ensure the application runs correctly before making changes.

---

### 5. Run Tests Before Submitting

```
npm run test
```

* Add tests for new components, hooks, or utilities
* Make sure all existing tests pass

---

### 6. Commit & Push Changes

```
git add .
git commit -m "Add: short description of your change"
git push origin feature/your-feature-name
```

---

### 7. Open a Pull Request

* Clearly describe **what** you changed and **why**
* Include screenshots or screen recordings for UI changes
* Reference related issues if applicable
